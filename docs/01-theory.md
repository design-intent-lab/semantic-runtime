# الجزء الأول: النظرية (الدستور)

لا يتغير إلا إذا ثبت خطأ فرضية جوهرية.

## 1. الرؤية
نبني طبقة دلالية تجعل عمليات التصميم ثلاثي الأبعاد قابلة للوصف، والتفسير، والقياس، والمراجعة.

## 2. المشكلة
برمجيات التصميم لا تحفظ كيف فكر الإنسان، بل ما أنشأه فقط.

## 3. المساهمة
تمثيل دلالي موحد لسير العمل الإبداعي.

## 4. المبادئ الخمسة
1. **قابل للشرح** — كل قرار في النظام يمكن تتبعه وتفسيره.
2. **قابل للإعادة** — نفس التمثيل ينتج نفس النتيجة.
3. **قابل للقياس** — قياس موضوعي (أرقام) وتفسيري (احتمالات بعرضها على المستخدم).
4. **قابل للدحض** — الفرضيات تُصاغ بطريقة تسمح بإثبات خطئها.
5. **القرار النهائي للإنسان** — النظام يوصي، المستخدم يقرر.

## 5. الوحدة الأساسية
**ثنائي التحول (Transformation Pair):** (Δ_Context , Δ_Result)
- السياق قبل التحول
- التحول نفسه (النوع والمعاملات)
- النتيجة المرصودة بعد التحول
